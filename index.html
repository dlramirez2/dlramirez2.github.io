The ABS system was developed for the Practicum course at UTEP in the Spring of 2021. I worked on this system with 4 other colleagues. Among other contributions, I was responsible for developing an algorithm that used the click images captured by ECELd to detect and produce coordinates of clicks. This algorithm uses a Python computer vision and the PyAutoGUI library to detect a small square in the image and calculate coordinates to reproduce the same click.

Web Application Security Workshop

This workshop was divided into two parts, both published writeups can be found in the repository. The first part of the workshop aims to introduce participants to basic web application security concepts such as Brute Force, XSS, and SQL Injection vulnerabilities. It also allows them to interact with two open-source security tools, Hydra-THC and sqlmap to identify and verify the vulnerabilities present in the system. The second part of the workshop is intended to provide participants an opportunity to experiment with remediation steps that can be taken once a vulnerability is found. Our goal was to show students that even if vulnerabilities are present, they can be remediated if identified.

This is my own notes page. Here I have gathered notes about software I have experience with or just information I would like to have easy access to.

  • Docker create container:
    • docker pull [imageName]
    docker run --name [containerName] -it imageName
  • Copy file to Container:
    • docker cp file.name [ContainerName]: /path/inside/container
  • Run Docker container:
    • docker start [containerName]
    docker exec -it [containerName] /bin/bash
    docker attach [containerName]
  • Docker Errors
    • **Docker can’t connect to docker daemon
    sudo groupadd docker
    sudo usermod -aG docker $(whoami)
    **Cannot connect to the Docker daemon at unix.... Is the docker daemon running?
    sudo service docker start
  • Docker Container List
    • docker ps -a
  • Docker Remove Container
    • docker rm container_name

  • Debian Install Volatility
    • sudo apt-get install volatility
  • Verify Volatility is installed
    • volatility -h
  • Get image OS Volatility:
    • ./vol.py imageinfo -f [/path/to/memory/dump]
  • View list of running processes
    • ./vol.py --profile=[ProfileOSName] pslist -f
  • Get Kernel Debugger Block address (identify correct profile):
    • ./vol.py --profile=[ProfileOSName] kdbgscan -g [/path/to/memory/dump]
  • Use Psscan for Malware Analysis (scan for hidden unlinked processes):
    • ./vol.py --profile=[ProfileOSName] psscan -f [/path/to/memory/dump]
  • Display all DLLs for X running process:
    • ./vol.py --profile=[ProfileOSName] dllist -p [processID] -f [/path/to/memory/dump]
  • Display all DLLs for all running processes:
    • ./vol.py --profile=[ProfileOSName] dllist -f [/path/to/memory/dump]
  • View Processes in tree form (PsTree):
    • ./vol.py --profile=[ProfileOSName] pstree -f [/path/to/memory/dump]
  • View commands typed locally or through backdoors:
    • ./vol.py --profile=[ProfileOSName] consoles -f [/path/to/memory/dump]
  • View version info of executables:
    • ./vol.py --plugins=contrib/plugins/ -f [/path/to/memory/dump] --profile-[ProfileOSName] verinfo
  • View TCP Connections:
    • ./vol.py --profile-[ProfileOSName] connections -f [/path/to/memory/dump]
  • View network connections (Wind XP & Win Server 2003):
    • ./vol.py --profile-[ProfileOSName] connscan -f [/path/to/memory/dump]
  • View network connections (Win & & beyond):
    • ./vol.py --profile-[ProfileOSName] netscan-f [/path/to/memory/dump]
  • View Active and terminated connections:
    • ./vol.py --profile-[ProfileOSName] connscan -f [/path/to/memory/dump]
  • View listening sockets (TCP & UDP):
    • ./vol.py --profile-[ProfileOSName] sockets -f [/path/to/memory/dump]
  • View lists of running services:
    • ./vol.py --profile-[ProfileOSName] svcscan -f [/path/to/memory/dump]

  • Get DBs Names
    • show databases;
  • Select Specific DB:
    • use [db_name];
  • Select Table Content:
    • SELECT * FROM [table_name];
  • Create Table:
    • CREATE TABLE [table_name] (Column_Name type(x),....);
  • Insert into Table:
    • INSERT INTO [table_name] VALUES ('X','Y',10...); (must match all columns in table)
    INSERT INTO [table_name] (column1,column2,column3) VALUES ('X','Y',10); (if only inserting in specific columns)
  • Delete Table:
    • DROP TABLE [table_name];
  • Delete Table:
    • DROP TABLE [table_name];
  • Create Table w/ Primary & Foreign Key:
    • CREATE TABLE [table_name] (Column_Name type(x),...., PRIMARY KEY (X), FOREIGN KEY (Y) REFERENCES table2(Y));
  • Column Properties:
    • [Column_Name] type(x) NOT NULL; (Primary Key)
     [Column_Name] type(x) DEFAULT 'XYZ'/#;
     [Column_Name] type(x) UNSIGNED; (NonNegative #s Only)
     [Column_Name] type(x) UNIQUE;
  • Procedures:
    • Call Procedure
     CALL simpleProc(@a); (@a is a variable)

     CREATE PROCEDURE sumpleProc(IN param1 TYPE)
     BEGIN
     SELECT COUNT(*) INTO param1 FROM table_name;
