Notes

This is my own notes page. Here I have gathered notes about software I have experience with.

  • Docker create container:
    • docker pull [imageName]
    docker run --name [containerName] -it imageName
  • Copy file to Container:
    • docker cp file.name [ContainerName]: /path/inside/container
  • Run Docker container:
    • docker start [containerName]
    docker exec -it [containerName] /bin/bash
    docker attach [containerName]
  • Docker Errors
    • **Docker can’t connect to docker daemon
    sudo groupadd docker
    sudo usermod -aG docker $(whoami)
    **Cannot connect to the Docker daemon at unix.... Is the docker daemon running?
    sudo service docker start
  • Docker Container List
    • docker ps -a
  • Docker Remove Container
    • docker rm container_name

  • Debian Install Volatility
    • sudo apt-get install volatility
  • Verify Volatility is installed
    • volatility -h
  • Get image OS Volatility:
    • ./vol.py imageinfo -f [/path/to/memory/dump]
  • View list of running processes
    • ./vol.py --profile=[ProfileOSName] pslist -f
  • Get Kernel Debugger Block address (identify correct profile):
    • ./vol.py --profile=[ProfileOSName] kdbgscan -g [/path/to/memory/dump]
  • Use Psscan for Malware Analysis (scan for hidden unlinked processes):
    • ./vol.py --profile=[ProfileOSName] psscan -f [/path/to/memory/dump]
  • Display all DLLs for X running process:
    • ./vol.py --profile=[ProfileOSName] dllist -p [processID] -f [/path/to/memory/dump]
  • Display all DLLs for all running processes:
    • ./vol.py --profile=[ProfileOSName] dllist -f [/path/to/memory/dump]
  • View Processes in tree form (PsTree):
    • ./vol.py --profile=[ProfileOSName] pstree -f [/path/to/memory/dump]
  • View commands typed locally or through backdoors:
    • ./vol.py --profile=[ProfileOSName] consoles -f [/path/to/memory/dump]
  • View version info of executables:
    • ./vol.py --plugins=contrib/plugins/ -f [/path/to/memory/dump] --profile-[ProfileOSName] verinfo
  • View TCP Connections:
    • ./vol.py --profile-[ProfileOSName] connections -f [/path/to/memory/dump]
  • View network connections (Wind XP & Win Server 2003):
    • ./vol.py --profile-[ProfileOSName] connscan -f [/path/to/memory/dump]
  • View network connections (Win & & beyond):
    • ./vol.py --profile-[ProfileOSName] netscan-f [/path/to/memory/dump]
  • View Active and terminated connections:
    • ./vol.py --profile-[ProfileOSName] connscan -f [/path/to/memory/dump]
  • View listening sockets (TCP & UDP):
    • ./vol.py --profile-[ProfileOSName] sockets -f [/path/to/memory/dump]
  • View lists of running services:
    • ./vol.py --profile-[ProfileOSName] svcscan -f [/path/to/memory/dump]
